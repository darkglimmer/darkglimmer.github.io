js模块化

Posted on 2018-03-07

webpack

在学习webpack过程中有几个我忽略和弄错的：

1.package.json中的script定义脚本命令。每当执行npm run，就会自动新建一个 Shell，在这个 Shell 里面执行指定的脚本命令。因此，只要是 Shell（一般是 Bash）可以运行的命令，就可以写在 npm 脚本里面。
比较特别的是，npm run新建的这个 Shell，会将当前目录的node_modules/.bin子目录加入PATH变量，执行结束后，再将PATH变量恢复原样。
这意味着，当前目录的node_modules/.bin子目录里面的所有脚本，都可以直接用脚本名调用，而不必加上路径。比如，当前项目的依赖里面有 Mocha，只要直接写mocha test就可以了。
在webpack进行打包时要注意修改为”build”: “webpack”

2.index.js是整个页面的全局，包含了其中模块化

3.运行“node_modules/.bin/webpack”出现：‘The ‘mode’ option has not been set. Set ‘mode’ option to ‘development’ or ‘production’ to enable defaults for this environment.’
检查：
设置src / index.js ，使其包含console.log(“Hello world”); 。
执行node_modules/.bin/webpack –mode development 。 Webpack将通过Node约定发现源文件。
检查dist / main.js。 您应该看到开始执行代码的webpack引导程序代码。 在bootstrap下面，你应该找到熟悉的东西。

4.module.exports后要接函数或者对象，不能直接输出

5.用了引用，在接入webpack入口时只需要加入引用后的模块，根模块不用引用。

为什么要模块化

最主要的作用就是产生独立的局部变量，减少全局变量：
1.减少“命名空间污染”，防止重复命名
2.便于维护
3.可用于其他文件

####实现模块模式的方法
从1999到现在有闭包模块化模式、Umd、YModules等多种方法，但是现在较流行的就是CommonJS、Amd、ES6
